# Design System & UX Guidelines - Wallet Safety Copilot

## Core Principles

- **Safety First**: Every interaction should prioritise user security.
- **Explain Before Warn**: Help users understand *what* is happening before telling them it's dangerous.
- **Warn Before Block**: Respect user autonomy unless the risk is extreme.
- **Plain Language**: Avoid crypto jargon (e.g., "approve," "gas," "bytes") unless explained or in Advanced mode.
- **Calm Visuals**: Use a professional, trustworthy color palette. Avoid alarmist red/yellow unless necessary.

## UX Rules

- High-risk actions must require deliberate, multi-step confirmation.
- Beginners should see more guided explanations; Advanced users see technical details via progressive disclosure.
- No silent auto-signing; every signature must have a human-readable preview.
- Use micro-animations to indicate "Security Copilot is analysing..." during simulations.

## Visual Identity

- **Safe State**: Calm greens, soft blues.
- **Warning/Suspicious**: Amber/Orange with clear "Why?" explanations.
- **High Risk/Dangerous**: Strong Red with "Block Recommendation" and "Confirm Anyway" (hidden/secondary).
